# Выборка полей json-объекта

```shell
jq '[.result[] | .type?] | unique' file.json
```

## Разбор по частям:

1. .result[]

- Обращается к объекту result в JSON
- [] означает "пройти по всем элементам объекта" (итерация)
- Превращает объект в поток значений

Пример:

```json
{
  "result": {
    "ID": {
      ...
    },
    "NAME": {
      ...
    },
    "SORT": {
      ...
    }
  }
}
```

.result[] превращает это в три отдельных объекта (значения полей ID, NAME, SORT)

2. | .type? (пайп и выбор поля)

- | передаёт каждый объект дальше
- .type берёт поле type из каждого объекта
- ? после .type означает "опционально" - если поля нет, не выдавать ошибку

3. [ ... ] (квадратные скобки)

- Собирает все результаты в массив
- Без скобок вы получите поток значений (каждое на новой строке при -r)

```shell
# Без скобок - поток
.result[] | .type?  → "integer", "string", "boolean"

# Со скобками - массив
[.result[] | .type?] → ["integer", "string", "boolean"]
```

4. | unique (финальная обработка)
- Принимает массив из предыдущего шага
- Удаляет дубликаты
- Сортирует результат

## Пошаговый пример:

Допустим, у нас JSON:

```json
{
  "result": {
    "ID": {
      "type": "integer"
    },
    "NAME": {
      "type": "string"
    },
    "STATUS": {
      "type": "string"
    },
    "ACTIVE": {
      "type": "boolean"
    }
  }
}
```

Как работает команда:

Шаг 1: .result[] → извлекает значения

```
{ "type": "integer" }
{ "type": "string" }
{ "type": "string" }
{ "type": "boolean" }
```

Шаг 2: | .type? → берёт поле type

```
"integer"
"string"
"string"
"boolean"
```

Шаг 3: [ ... ] → собирает в массив

```
["integer", "string", "string", "boolean"]
```

Шаг 4: | unique → удаляет дубликаты и сортирует

```
["boolean", "integer", "string"]
```
